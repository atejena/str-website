## Component Architecture

### Server Components (Data Fetching)
- `page.tsx` - Main page, orchestrates all sections
- `OurOfferings.tsx` - Fetches gym classes
- `GalleryPreview.tsx` - Fetches gallery images
- `GoogleMaps.tsx` - Fetches map embed URL
- `Footer.tsx` - Receives social links as props

### Client Components (Interactivity)
- `GetStartedForm.tsx` - Form handling, state management
- `TestimonialsScrolling.tsx` - Infinite scroll animation
- `Header.tsx` - Mobile menu, scroll state
- `MobileNav.tsx` - Mobile navigation overlay
- `ThemeToggle.tsx` - Light/dark mode toggle

### Server Actions
- `submitContactForm()` - Validates and saves form data

## Responsive Behavior

### Mobile (< 640px)
- Single column layouts
- Stacked cards
- Full-width buttons
- Hamburger menu

### Tablet (640px - 1023px)
- 2-column grids
- Condensed spacing
- Visible navigation (some routes)

### Desktop (1024px+)
- Multi-column layouts
- Full navigation bar
- Larger typography
- More whitespace

## Data Flow

```
Database (Supabase)
       ↓
Server Components (Fetch on each request)
       ↓
Props → Client Components (Interactive UI)
       ↓
Server Actions → Database (Form submissions)
```
